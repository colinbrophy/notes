# Tags

Tags are for **properties and states** of notes (not what the note *is*). Use them for cross-cutting filters and workflow.
Use the same vocabulary whether tags appear inline (e.g. `#draft`) or in frontmatter (e.g. `tags: [draft, ai-written]`).
Keep the set small; if a new tag is needed, document it here.

## Workflow

- `#todo`: actionable tasks/checklists to work through.
- `#open-questions`: unresolved questions or uncertainties worth revisiting.
- `#draft`: incomplete write-up that needs shaping.

## Authorship / provenance

- `#ai-written`: written primarily by an AI assistant; treat as rough and verify/edit before relying on it.
- `#human-written`: written directly by hand; use only when authorship matters.
- `#aidump`: AI-generated or AI-captured material kept mainly as a rough dump/reference, not a finished note.

## Note types

- `#index`: navigation / map-of-content notes (lists of links).
- `#source`: primarily sourced material (quotes, transcripts, extracts).
- `#quote`: contains substantial quoted text (usually with context).

## Storage

- `#archive`: imported/backlog material kept for reference but not maintained.
